# Night Shift Dashboard Design System

## Direction

- Style: PS5-inspired Dashboard OS for a personal productivity console.
- Feel: premium, cinematic, focused, and fast to navigate.
- User context: a single user moving between tasks, habits, notes, and planning views with minimal friction.

## Domain Anchors

- Console home screen
- Focus hub
- Session resume
- Destination rail
- Ambient status layer
- Activity pulse

## Palette

- Canvas: deep navy and midnight blue backgrounds.
- Accent: electric blue for active states, primary actions, and focus glow.
- Text: cool white for primary content, mist blue-gray for secondary text.
- Surfaces: translucent blue-black glass panels.
- Semantic colors: keep success and destructive states clear but slightly cooled to fit the palette.

## Depth Strategy

- Use layered surfaces with subtle borders and soft large shadows.
- Keep borders low contrast and let active glow communicate emphasis.
- Prefer glass panels and ambient highlights over heavy separators.

## Typography

- Display font: geometric, futuristic, and clean for headings.
- Body font: modern sans-serif with strong readability.
- Headlines should feel deliberate and slightly condensed through tracking.

## Spacing

- Base unit: 4px.
- Common rhythm: 12px, 16px, 24px, 32px.
- Large hero and shell zones should breathe more than dense list rows.

## Navigation Patterns

- Sidebar acts as a destination rail with large rounded targets.
- Active destination uses brighter surface, stronger contrast, and soft blue shadow.
- Top bar acts like a console status strip with page title, live state, and quick actions.

## Component Patterns

- Cards: rounded glass surfaces with subtle top-light sheen.
- Buttons: pill-shaped, slightly elevated, with the primary action glowing blue.
- Inputs: inset control surfaces with rounded geometry and restrained focus rings.
- Dialogs: large blurred overlays with premium panel treatment.
- Hero sections: feature a "resume session" feel with action shortcuts and stats.

## Motion

- Use quick slide and fade transitions.
- Hover and active states should lift slightly, never bounce.
- Motion should reinforce focus, not become decorative noise.

## Avoid

- Flat admin-dashboard styling
- Harsh white cards
- Multiple accent hues competing for attention
- Dense visual clutter in the shell
- Overly playful motion
